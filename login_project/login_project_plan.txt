1. form 태그의 역할은 무엇인가?  >    clear
    > 특정 서버에 데이터를 전송하기 위한 태그
    > form태그안에서 button type submit 과 input type ='submit'의 차이는?
        >> type submit : submit 버튼을 클릭하면 폼 데이터를 <form> 요소의 action속성에 정의된 웹페이지 에 전송된다.
        >> 기본적으로 button type default는 submit이다.
        >> 둘의 기능적인 부분은 비슷하다.
        >> 하지만 input은 버튼안에 text만 담을 수 있지만 button은 다른 컨텐츠(이미지 등)를 담을 수 있다. 

    > 회원가입에서 데이터베이스로 데이터를 보내기위해 form태그 action 속성에 데이터베이스 url을 입력해줘야함.
    > 또한 protected 페이지로 데이터를 보내 이를 확인하여 authentication을 실행한다.
    (즉 login 페이지에서도 form 태그를 사용하자.)

    > form에서 action의 주소로 보낸 데이터는 해당 주소에서 어떻게 전달받을 수 있나?
    input에 name 속성을 할당하여서 서버에서 해당 name으로 input value를 조회할 수 있다.

    또는 protected 컴포넌트에서 query string 값으로도 받을 수 있다.
